# 🛡️ Админ Панель - Полное Руководство по Настройке

## 📋 Что было сделано?

Была создана полнофункциональная администраторская панель с возможностью:

✅ **Управление пользователями**
- Просмотр списка всех пользователей
- Изменение ролей (пользователь ↔ администратор)
- Удаление пользователей

✅ **Управление консультантами**
- Просмотр списка консультантов
- Включение/отключение доступности
- Просмотр рейтингов и информации

✅ **Модерация контента**
- Просмотр и удаление отзывов
- Просмотр истории чатов
- Статистика по активности

✅ **Статистика и аналитика**
- Всего пользователей
- Активные консультанты
- Общее количество чатов
- Активные чаты
- Опубликованные отзывы

## 🚀 Шаг 1: Создание первого администратора

### Вариант A: Через Supabase Console (Рекомендуется)

1. Зайди на https://app.supabase.com
2. Выбери свой проект
3. Перейди на **SQL Editor** (левое меню)
4. Выполни этот запрос (замени email на свой):

```sql
UPDATE users 
SET role = 'admin' 
WHERE email = 'your_email@example.com';
```

5. Нажми **Run** или Ctrl+Enter
6. ✅ Готово! Теперь ты администратор

### Вариант B: Через Table Editor

1. В Supabase откройте **Table Editor**
2. Откройте таблицу **users**
3. Найдите своего пользователя
4. Измените поле `role` с `user` на `admin`
5. Сохраните

## 📍 Шаг 2: Доступ к админке

### Как попасть в админ панель?

**Для администраторов:**
1. Зайди на сайт и авторизуйся

3. На боковой панели появится кнопка "🛡️ Админ панель"

## 🆘 Troubleshooting

### "Админ кнопка не видна в профиле"

**Решение:**
```
1. Проверь, что роль установлена на 'admin' в Supabase
2. Перезагрузи страницу (F5)
3. Очисти кэш браузера (Ctrl+Shift+Delete)
4. Перезагрузись (выход/вход)
```

### "Админка не загружается"

**Решение:**
```
1. Проверь подключение к Supabase (консоль F12)
2. Убедись, что таблицы существуют в БД
3. Проверь переменные окружения (.env.local)
4. Посмотри логи в браузере (F12 → Console)
```

### "Я админ, но не вижу данные"

**Решение:**
```
1. Убедись, что таблицы не пустые (можешь проверить в Supabase)
2. Проверь, есть ли ошибки в консоли браузера
3. Перезагрузи приложение
4. Проверь, что у таблиц правильные имена:
   - users
   - companions
   - reviews
   - chats
```

## 📚 Справка

### Роли пользователей в системе:

| Роль | Доступ | Возможности |
|------|--------|------------|
| `user` | Профиль, поиск, чаты | Поиск консультантов, переписка, отзывы |
| `admin` | Админ панель + все как у user | Управление всеми данными, модерация |

### SQL команды для администраторов:

```sql
-- Просмотр всех админов
SELECT * FROM users WHERE role = 'admin';

-- Просмотр всех пользователей
SELECT id, name, email, role FROM users;

-- Количество пользователей
SELECT COUNT(*) as total_users FROM users;

-- Активные чаты
SELECT COUNT(*) as active_chats FROM chats WHERE status = 'active';

-- Удалить админские права
UPDATE users SET role = 'user' WHERE email = 'admin@example.com';
```
