## 1. Получение Supabase учетных данных

### Шаг 1: Перейти на https://supabase.com

1. Нажми **"Sign In"** (если уже зарегистрирован) или **"Sign Up"** (для регистрации)
2. Авторизуйся через GitHub, Google или Email

### Шаг 2: Создать новый проект

1. Перейди на вкладку **"Projects"** в левом меню
2. Нажми **"New project"**
3. Заполни форму:
   - **Project Name**: `ryadom` (или любое название)
   - **Database Password**: Придумай сильный пароль (запомни!)
   - **Region**: Выбери **Europe (EU-West-1)** или ближайший регион
4. Нажми **"Create new project"**
5. Ожидай 2-5 минут, пока проект создастся

### Шаг 3: Получить API ключи

После создания проекта:

1. Нажми на свой проект, чтобы открыть панель управления
2. Слева выбери **Settings** (⚙️)
3. Выбери вкладку **API**
4. Скопируй два значения:
   - **Project URL** (выглядит как: `https://xxx.supabase.co`)
   - **anon public** ключ (публичный API ключ)

**Пример:**
```
Project URL: https://fisamcoxqplzqzfimwfx.supabase.co
Anon Key: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 2. Заполнение переменных окружения

1. Открой файл `.env.local` в корне проекта
2. Замени плейсхолдеры на свои значения:

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT_ID.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=YOUR_ANON_KEY_HERE
```

**Пример (с реальными значениями):**
```env
VITE_SUPABASE_URL=https://fisamcoxqplzqzfimwfx.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

3. Сохрани файл
4. Перезагрузи браузер (F5)

## 3. Создание таблиц в Supabase

### Способ 1: Через SQL Editor (Рекомендуется)

1. В Supabase панели перейди на **SQL Editor** (левое меню)
2. Нажми **New query** (или **+** вверху)
3. Скопируй весь код из файла `SUPABASE_SCHEMA.sql` этого проекта
4. Вставь его в SQL Editor
5. Нажми **Run** (или Ctrl+Enter) в левом верхнем углу
6. Жди, пока выполнится (должны создаться все таблицы без ошибок)

### Способ 2: Вручную через Table Editor

Если SQL Editor не работает:

1. В левом меню Supabase выбери **Table Editor**
2. Нажми **Create a new table**
3. Создай таблицы согласно структуре в `SUPABASE_SCHEMA.sql`

> 📌 **Рекомендуется использовать Способ 1** - это намного быстрее.

## 4. Добавление sample данных (опционально)

Для тестирования можно добавить sample собеседников:

1. Открой `SUPABASE_SCHEMA.sql`
2. Внизу файла найди раздел **SAMPLE DATA**
3. Раскомментируй SQL код (удали `--` в начале строк)
4. Выполни этот SQL в **SQL Editor** в Supabase

После этого появятся 3 тестовых собеседника.

## 5. Проверка работы

### 5.1 Проверить создание таблиц

1. В Supabase панели, слева выбери **Table Editor**
2. Проверь, что видишь все таблицы:
   - `users`
   - `companions`
   - `companion_topics`
   - `chats`
   - `messages`
   - `favorites`
   - `reviews`
   - `reports`

### 5.2 Проверить авторизацию

1. Открой приложение в браузере: `http://localhost:5173`
2. Нажми **"Регистрация"**
3. Заполни форму (любой email и пароль)
4. Нажми **"Создать аккаунт"**

Если появилась ошибка:
- Проверь, правильно ли скопирована URL и ключ в `.env.local`
- Убедись, что в Supabase включена **Email Auth** (Settings → Authentication)
- Перезагрузи браузер (F5)

### 5.3 Проверить чтение данных

1. После входа перейди на **"Начать поиск"** или на страницу поиска собеседников
2. Должна загрузиться список собеседников из Supabase (если они добавлены)

## Важные замечания

### ⚠️ Безопасность

1. **Не коммитьте `.env.local`** - добавь в `.gitignore`:
   ```
   .env.local
   ```

2. **Используй публичный ключ (anon key)** для фронтенда - это безопасно благодаря RLS политикам

3. **Row Level Security (RLS)** включена по умолчанию в SQL скрипте - это защитит данные пользователей

### 📝 Что изменилось

| Компонент | Было | Стало |
|-----------|------|--------|
| **Authentication** | Локальный backend | Supabase Auth |
| **Пароли** | Хранились в БД | Supabase шифрует |
| **Сообщения** | Mock данные | Supabase |
| **Профили** | Mock данные | Supabase |
| **Собеседники** | Mock данные | Supabase |
| **Backend** | Node.js Express | Не требуется |

### 🔧 Локальный Backend

Локальный backend **больше не используется**. Приложение работает исключительно с Supabase.

Если хочешь удалить его:
```bash
rm -rf backend
```

## Решение проблем

### ❌ "Supabase credentials not found"

**Решение:**
- Убедись, что `.env.local` существует
- Проверь, что переменные заполнены правильно
- Перезагрузи браузер (F5)

### ❌ "Failed to connect to Supabase"

**Решение:**
- Скопируй URL правильно (без лишних пробелов)
- Проверь интернет соединение
- Убедись, что Supabase проект активен

### ❌ Таблицы не создались

**Решение:**
- Убедись, что SQL код из `SUPABASE_SCHEMA.sql` полный
- Проверь логи в SQL Editor
- Попробуй создать таблицы вручную через Table Editor

### ❌ Авторизация не работает

**Решение:**
- Проверь Email Auth в Settings → Authentication
- Убедись, что включены подтверждение email (если нужно)
- Проверь консоль браузера (F12) на ошибки

---
