## ⚠️ Что нужно сделать:

### Шаг 1: Выполнить SQL схему в Supabase

1. Перейди на https://app.supabase.com
2. Выбери проект `fisamcoxqplzqzfimwfx`
3. В левом меню выбери **SQL Editor**
4. Нажми **New query**
5. Открой файл `SUPABASE_SCHEMA.sql` из этого проекта
6. Скопируй весь код и вставь в SQL Editor
7. Нажми **Run** (или Ctrl+Enter)

**Это создаст все необходимые таблицы и RLS политики.**

### Шаг 2: Включи Email Auth в Supabase

1. В Supabase выбери **Authentication** → **Providers**
2. Убедись, что **Email** включен
3. Если нужно, отключи требование подтверждения email для тестирования:
   - Перейди в **Settings** → **Email**
   - Отключи опцию "Confirm email" (опционально для тестирования)

### Шаг 3: Протестируй регистрацию

1. Открой приложение: http://localhost:5173
2. Нажми на вкладку **Регистрация**
3. Заполни форму:
   - Полное имя: любое имя (например, "Иван Иванов")
   - Email: любой email (например, test@example.com)
   - Пароль: минимум 6 символов
   - Подтверди пароль
   - Согласись с условиями
4. Нажми **Создать аккаунт**

**Если всё работает:**
- ✅ Пользователь создается в `auth.users` (Supabase Auth)
- ✅ Профиль пользователя сохраняется в таблицу `users`
- ✅ Приложение переходит на страницу профиля

**Если ошибка:**
- Проверь консоль браузера (F12) - там будет подробная ошибка
- Убедись, что SQL схема выполнилась без ошибок
- Проверь, что Email Auth включен в Supabase
- Убедись, что переменные в `.env` скопированы правильно

## 📝 Что происходит при регистрации:

1. **Frontend** (Vue) отправляет email и пароль
2. **Supabase Auth** создает пользователя в системе аутентификации
3. **Frontend** берет ID созданного пользователя
4. **Frontend** вставляет профиль в таблицу `users` с этим ID
5. **RLS политики** проверяют, что пользователь может только редактировать свой профиль
6. **Приложение** сохраняет пользователя в Vuex/localStorage и переходит на профиль

## 🔐 Безопасность

- ✅ Пароли хранятся только в Supabase Auth (зашифрованы)
- ✅ RLS политики защищают данные других пользователей
- ✅ Только аутентифицированные пользователи могут создавать/редактировать свои профили
- ✅ Email уникален в системе (UNIQUE constraint)

## 🚀 Дальнейшие шаги

После успешного входа/регистрации:
